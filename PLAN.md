# แผนงาน MVP — Recycle Fund (ระบบรับซื้อของเก่าออนไลน์)

> เอกสารนี้คือแผน MVP ครบ flow สำหรับพัฒนาให้พร้อมขาย/ขึ้น Production
> อัปเดตล่าสุด: 2026-07-04

---

## 1. ภาพรวมธุรกิจ (Business Overview)

**ปัญหา:** ครัวเรือน/ร้านค้ามีของเก่า (ขวด กระดาษ เหล็ก พลาสติก) แต่ไม่สะดวกขนไปขาย ส่วนซาเล้ง/ร้านรับซื้อก็หา "ของ" และวางแผนเส้นทางรับของยาก

**ทางออก:** แพลตฟอร์มจับคู่ **ผู้ขาย (มีของ)** กับ **ผู้ซื้อ/คนขับ (มารับถึงบ้าน)** พร้อมระบบ **จองรอบเข้ารับ + ติดตามสถานะ** และ **ระบบลุ้นรางวัลกระตุ้นการขาย**

**กลุ่มผู้ใช้:**
- ผู้ขาย: ครัวเรือน, ร้านค้า, ออฟฟิศ, คอนโด
- ผู้ซื้อ/คนขับ: ซาเล้ง, ร้านรับซื้อของเก่า, รถรับซื้อ

**โมเดลรายได้ (เลือกได้หลายทาง):**
| ช่องทาง | รายละเอียด |
|---|---|
| ค่าคอมมิชชัน | หัก % จากยอดรับซื้อของผู้ซื้อ (เช่น 3–5%) |
| ค่าสมาชิกผู้ซื้อ | รายเดือน สำหรับคนขับที่รับงานไม่จำกัด |
| โฆษณา/สปอนเซอร์ | แบรนด์รีไซเคิล, ธนาคารขยะ, CSR |
| ขายระบบ (SaaS) | ขายให้เทศบาล/ร้านรับซื้อรายใหญ่ทำแบรนด์ตัวเอง (white-label) |

## 2. ผู้ใช้งาน & บทบาท (Roles)

| บทบาท | ทำอะไรได้ |
|---|---|
| **ผู้ขาย (Seller)** | ลงรายการของเก่า, เลือกจุดรับ, จองวัน-เวลาให้มารับ, ติดตามสถานะ, ดูรายได้, ลุ้นรางวัล, ต่อ LINE OA |
| **ผู้ซื้อ/คนขับ (Buyer)** | จัดตารางรอบเข้ารับ (วัน/โซน/จำนวน), รับ-ปฏิเสธงาน, อัปเดตสถานะ (กำลังไป/สำเร็จ), บันทึกยอดจริง |
| **แอดมิน (Admin)** | อัปเดตราคากลาง, ประกาศผลรางวัล, จัดการผู้ใช้/รายงาน (เฟสถัดไป) |

## 3. User Flows หลัก

### 3.1 Flow ผู้ขาย (ขายของเก่า)
```
ลงทะเบียน/เข้าสู่ระบบ → หน้าแรก → "สร้างรายการ"
   → เลือกวัสดุ + ปริมาณโดยประมาณ (เห็นราคาต่อหน่วย → ประเมินยอดได้)
   → ปักหมุดจุดรับ (Google Map / ตำแหน่งปัจจุบัน) + บ้านเลขที่ + จุดสังเกต
   → กรอกผู้ติดต่อ + เบอร์
   → จองรอบเข้ารับ (เลือกจากตารางว่างของผู้ซื้อในโซน)
   → ส่งงาน (submitted)
   → รับแจ้งเตือนผ่าน LINE OA เมื่อสถานะเปลี่ยน
   → เมื่อสำเร็จ: ได้รายได้ + สิทธิ์ลุ้นรางวัล (100 บาท = 1 สิทธิ์)
```

### 3.2 Flow ผู้ซื้อ/คนขับ (รับซื้อ)
```
ลงทะเบียนคนขับ → จัดตารางรอบเข้ารับ (วัน + ช่วงเวลา + โซน + จำนวนที่รับได้)
   → เห็นงานที่ถูกจอง/งานเปิด → คอนเฟิร์ม หรือ ยกเลิก
   → วันงาน: อัปเดตสถานะ "กำลังไปรับ" → ไปถึง ชั่งของจริง
   → บันทึกยอดจริง → ปิดงาน "สำเร็จ"
   → ผู้ขายได้รับเงิน + สิทธิ์รางวัลอัตโนมัติ
```

### 3.3 Flow รางวัล (Engagement)
```
ทุกการขายสำเร็จ ≥100 บาท → ออกเลขสิทธิ์อัตโนมัติ
   → สะสมทั้งเดือน → สิ้นเดือนแอดมินสุ่ม/ประกาศเลขที่ออก
   → หน้าประกาศรางวัล (ธีม Gold/Black) แสดงเลข + ผู้ได้รับ
```

### 3.4 Status Machine ของงาน
```
submitted ──confirm──▶ confirmed ──start──▶ en_route ──finish──▶ completed
    │                      │                    │
    └──────cancel──────────┴────────cancel──────┘──▶ cancelled
```

## 4. Data Model (ตารางหลัก)

| `job_status_history` | job_id, status, note, at |
| `schedule_slots` | id, buyer_id, date, slot, area, capacity, booked |
| `reward_tickets` | id, number, user_id, month, from_job_id |
| `reward_draws` | month, prize_name, winning_number, winner_name, status, announced_at |

## 5. สถาปัตยกรรม & เทคโนโลยี

| ชั้น | เทคโนโลยี | เหตุผล |
|---|---|---|
| Frontend | **Next.js 15 (App Router) + TypeScript + Tailwind** | เว็บ production, mobile-first, deploy ง่าย |
| Backend/DB | **Supabase** (Postgres + Auth + Realtime + Storage + RLS) | ครบในตัว, มี Auth เบอร์โทร OTP + อีเมล, ปลอดภัยด้วย RLS |
| แผนที่ | **Google Maps JS API** | ปักหมุด/ตำแหน่งปัจจุบัน |
| แจ้งเตือน | **LINE Login + Messaging API (OA)** | คนไทยใช้ LINE, push แจ้งสถานะงานถึงผู้ขาย |
| Deploy | **Vercel** + Supabase Cloud | CI/CD อัตโนมัติ, ฟรีทดสอบ, สเกลได้ |

**หมายเหตุสำคัญ (สำหรับ MVP เดโม):** เวอร์ชันนี้มี **Data Layer แบบ localStorage** ในตัว ทำให้แอป *กดใช้งานได้ครบทุก flow ทันทีโดยไม่ต้องตั้งค่า backend* — เหมาะสำหรับสาธิต/พรีเซนต์ให้ลูกค้า แล้วค่อยสลับไปใช้ Supabase จริงตาม Phase 1 (โครงโค้ด Supabase เตรียมไว้แล้วใน `src/lib/supabase/`)

## 7. Roadmap แบ่งเป็น Phase

### ✅ Phase 0 — MVP Prototype (สัปดาห์นี้) « โปรเจกต์นี้ »
- โครงแอปครบทุกหน้า + flow, data layer localStorage, ธีม/ดีไซน์, สาธิตได้จริง

### Phase 1 — Backend จริง (1–2 สัปดาห์)
- ต่อ Supabase: Auth (อีเมล + เบอร์ OTP ผ่าน SMS provider), ตาราง + RLS, migrate data layer → Supabase
- ยืนยันตัวตนคนขับ (KYC เบา ๆ: บัตร ปชช./รูป)

### Phase 2 — Integrations (1–2 สัปดาห์)
- Google Maps จริง (ปักหมุด + คำนวณโซน/ระยะ)
- LINE Login เชื่อมบัญชี + LINE Messaging API push แจ้งเตือนสถานะ
- Realtime ติดตามสถานะงานสด

### Phase 3 — เงิน & แอดมิน (2 สัปดาห์)
- ระบบจ่ายเงิน/ตัดยอด (PromptPay/โอน), สรุปยอด payout คนขับ
- Admin panel: อัปเดตราคากลาง, สุ่ม/ประกาศรางวัล, จัดการผู้ใช้-รายงาน

### Phase 4 — Production Hardening & Launch (1 สัปดาห์)
- ทดสอบโหลด, security review, PDPA, toc/privacy, monitoring, backup, โดเมน, LINE OA verified
